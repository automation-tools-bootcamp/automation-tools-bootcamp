Introduction to Packer


This session is nearly the antithesis for the Chef topic that was just covered. Short and sweet. But you can't let that fool you, Packer is pow. er. ful.

"Packer is an open source tool for creating identical machine images for multiple platforms from a single source configuration." Packer is open source and watched over by creator Mitchell Hashimoto. Remember him?

Official Intro to Packer
And that is about it. Packer embodies the *nix philosophy. Do one job and do that one job well.

How to build your image

Packer is here to create OS images of your server platforms. "Golden Images" is used at times to describe this.

You need three things to be get running with Packer.

  1. Install Packer
  2. Create a json file defining the image you'll be creating
  3. Patience - you're working with some big ol' files here.

Advanced Packer

Yes, Packer is just going to build images for you. But there are mulitple paths to the destination.

Workout

We are going to complete the following actions

  1. Install Packer on *your* machine
  2. Go in to the packer directory of this git repo and packer validate amazon-example.json .
  3. Create a Free AWS account.
  4. Set two environment variables AWS_ACCESS_KEY and AWS_SECRET_KEY from your AWS account.
  5. Build your version of the AMI packer build amazon-example.json
  6. Delete the AMI from your AWS Console.
